Программа для занесения данных о проекте в БД.

Реализована программа для занесения данных о проектах в базу данных и дальнейшей работы
с этими данными, со следующими возможностями:

— Возможность создавать/просматривать/редактировать/удалять информацию о проектах.
— Возможность создавать/просматривать/редактировать/удалять информацию о работниках.
— Возможность добавлять и удалять работников c проекта (один сотрудник может работать
одновременно на нескольких проектах, и на одном проекте может работать несколько человек).
— Для просмотра проектов предусмотреть различные способы фильтрации (по диапазону даты
начала, по приоритетам, и т.п.) и сортировки (по основным полям).
Необходимая для хранения информация:
— название проекта,
— название компании-заказчика,
— название компании-исполнителя,
— данные о сотруднике (имя, фамилия, отчество, e-mail человека),
— данные о руководителе проекта,
— данные об исполнителях проекта,
— даты начала и окончания проекта,
— приоритет проекта (целочисленное значение >=0),
— текстовый комментарий.


В решении реализована архитектура n-tier:
1) Уровень доступа к данным реализован в виде Wcf-приложения(DALWcfApp) содержащего Wcf data Service, который использует Entity Framework 6.x для доступа к базе данных. В нем же реализована модель данных сущностей.

2) Уровень доступа к бизнесслогике представляет собой три проекта(ILogicLibrary, LogicLibrary, ClientEntities). В них соответственно располагаются интерфейсы бизнес-логики, реализация бизнес-логики и клиентская модель данны, и библиотека с сущностями клиентской модели данных.

3) Уровень доступа к отображению представляет собой клиентское приложение(ViewLayerWpfApp), реализующее шаблон MVVM (все классы модели располагаются в слое бизнес-логики).

В решении частично реализована аутентификация и авторизация. Выполнено автоматическое добавление пользователя "Admin" с паролем "123" и двух ролей:
"Admin" и "User", в уровне доступа к данным.

Начата реализация многопоточности(с помощью класса Task и пула потоков) для осуществления асинхронных запросов к БД(В модели отображения проектов 
реализовано асинхронное взаимодействие с бэкэндом, в модели отображения сотрудников - только загрузка данных), 
с сохранением возможности реагирования UI приложения и отображения анимации при загрузке.


В ПО реализована возможность добавления/редактирования/удаления данных прямо в таблицах путём использования кнопок [Добавить] и [Удалить] под таблицей и использования контекстного меню таблиц с пунктами [Добавить] и [Удалить]. После добавления и редактирования данных строки подсвечиваются серым цветом.
Для сохранения всех внесённых изменений необходимо нажать кнопку [Сохранить изменения], для отмены - [Отменить изменения].

Фильтр данных работает как в режиме подгрузки данных из базы данных, так и с данными, уже загруженными в клиентское приложение. 
Изменение режима происходит при помощи установки или отмены галочки [Подгружать данные из базы данных]. 
В режиме подгрузкиданных фильтрация работет только по нажатию кнопки [Отобразить].
